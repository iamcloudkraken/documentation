# Amazon S3 Source

The Amazon S3 source reads files from an S3 bucket (or S3-compatible object store) and emits each file's records as a stream. It supports both event-driven triggering (via S3 Event Notifications) and periodic polling.

Delivery guarantee: **at-least-once**. Files that fail mid-processing are retried based on the configured retry policy.

## Trigger Modes

### Event notifications (recommended)

STREAMINGPLUS subscribes to an SQS queue that receives S3 Event Notifications. New files are processed within seconds of being written to the bucket.

```yaml
trigger:
  type: event-notification
  eventNotification:
    sqsQueueUrl: https://sqs.us-east-1.amazonaws.com/123456789012/s3-events-queue
    sqsRegion: us-east-1
    visibilityTimeout: 300s
    waitTimeSeconds: 20
```

Set up S3 â†’ SQS event notifications:

```bash
# Create SQS queue
aws sqs create-queue --queue-name s3-events-queue

# Add S3 notification configuration
aws s3api put-bucket-notification-configuration \
  --bucket my-data-bucket \
  --notification-configuration '{
    "QueueConfigurations": [{
      "QueueArn": "arn:aws:sqs:us-east-1:123456789012:s3-events-queue",
      "Events": ["s3:ObjectCreated:*"],
      "Filter": {
        "Key": {"FilterRules": [{"Name": "prefix", "Value": "data/"}]}
      }
    }]
  }'
```

### Poll mode

STREAMINGPLUS periodically lists the bucket and processes any new objects since the last checkpoint. Simpler to set up but has higher latency.

```yaml
trigger:
  type: poll
  poll:
    interval: 60s
    pathGlob: "data/year=*/month=*/day=*/*.parquet"
```

:::note
In poll mode, STREAMINGPLUS tracks processed files using a checkpoint stored in its state backend. Objects are processed in last-modified-time order within each poll cycle. Avoid overwriting files in place â€” append new files or use versioned paths.
:::

## Authentication

### IRSA (recommended)

```yaml
auth:
  type: irsa
  roleArn: arn:aws:iam::123456789012:role/streamingplus-s3-reader
```

Required IAM permissions:

```json
{
  "Effect": "Allow",
  "Action": [
    "s3:GetObject",
    "s3:ListBucket",
    "s3:GetBucketLocation"
  ],
  "Resource": [
    "arn:aws:s3:::my-data-bucket",
    "arn:aws:s3:::my-data-bucket/*"
  ]
}
```

### Static credentials

```yaml
auth:
  type: static
  accessKeyId:
    secretRef:
      name: s3-credentials
      key: access-key-id
  secretAccessKey:
    secretRef:
      name: s3-credentials
      key: secret-access-key
```

## Full YAML Example

```yaml
apiVersion: streamingplus.io/v1
kind: StreamingPipeline
metadata:
  name: s3-clickstream-loader
  namespace: data-platform
spec:
  source:
    type: s3
    s3:
      bucket: my-clickstream-data
      region: us-east-1
      prefix: events/raw/
      suffix: .json
      valueFormat: json
      compressionType: gzip
      recursive: true
      partitionDiscovery: true

      trigger:
        type: event-notification
        eventNotification:
          sqsQueueUrl: https://sqs.us-east-1.amazonaws.com/123456789012/clickstream-s3-events
          sqsRegion: us-east-1
          visibilityTimeout: 600s

      auth:
        type: irsa
        roleArn: arn:aws:iam::123456789012:role/streamingplus-s3-reader

      errorHandling:
        maxRetries: 3
        retryBackoff: exponential
        deadLetterBucket: my-clickstream-data-dlq
        deadLetterPrefix: failed/

      processing:
        parallelism: 4
        batchSize: 1000
```

## S3-Compatible Stores

Point the S3 source at any S3-compatible object store (MinIO, Ceph, Wasabi) by overriding the endpoint:

```yaml
s3:
  bucket: my-bucket
  region: us-east-1
  endpoint: https://minio.internal.example.com
  pathStyleAccess: true
  auth:
    type: static
    accessKeyId:
      secretRef:
        name: minio-credentials
        key: access-key
    secretAccessKey:
      secretRef:
        name: minio-credentials
        key: secret-key
```

## CLI Quickstart

```bash
sp pipelines create s3-clickstream-loader \
  --source s3 \
  --bucket my-clickstream-data \
  --region us-east-1 \
  --prefix events/raw/ \
  --format json \
  --trigger event-notification \
  --sqs-url https://sqs.us-east-1.amazonaws.com/123456789012/clickstream-s3-events \
  --role-arn arn:aws:iam::123456789012:role/streamingplus-s3-reader \
  --namespace data-platform
```

Check processing progress:

```bash
sp sources status s3-clickstream-loader
```

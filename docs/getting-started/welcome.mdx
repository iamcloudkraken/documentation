# Welcome to STREAMINGPLUS

STREAMINGPLUS is a **cloud-native streaming infrastructure platform** designed to simplify the deployment, management, and scaling of real-time data pipelines across any cloud environment. Whether you are running Kafka clusters, Flink jobs, Pulsar topics, or custom stream processors, STREAMINGPLUS gives your team a unified control plane to govern it all — without lock-in.

Built for platform engineering teams and data infrastructure teams, STREAMINGPLUS abstracts the complexity of multi-cloud streaming infrastructure behind a clean API, a powerful CLI, and a GitOps-native workflow engine.

---

## What is STREAMINGPLUS?

STREAMINGPLUS is structured around three ideas:

1. **Declarative infrastructure** — Describe your streaming topology in YAML. STREAMINGPLUS reconciles the world to match your intent.
2. **Multi-cloud by default** — Run pipelines across AWS, GCP, Azure, or on-premises Kubernetes clusters from a single control plane.
3. **Operator-grade reliability** — Built on Kubernetes-style controllers, mTLS-secured agents, and an OPA policy engine that enforces governance without slowing teams down.

## How STREAMINGPLUS Works

At a high level, STREAMINGPLUS sits between your application code and your target cloud providers:

```
┌────────────────────────────────────────────────────────────────┐
│                        Your Code / GitOps Repo                  │
│   (YAML manifests, pipeline definitions, environment configs)   │
└────────────────────────┬───────────────────────────────────────┘
                         │
                         ▼
┌────────────────────────────────────────────────────────────────┐
│                       STREAMINGPLUS                             │
│                                                                │
│  ┌─────────────┐  ┌──────────────┐  ┌────────────────────┐   │
│  │  Control    │  │   GitOps     │  │   Policy Engine     │   │
│  │  Plane API  │  │  Reconciler  │  │   (OPA)             │   │
│  └─────────────┘  └──────────────┘  └────────────────────┘   │
│                                                                │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │              sp-agent (Data Plane)                       │  │
│  │  Runs as a DaemonSet on each cluster                     │  │
│  └─────────────────────────────────────────────────────────┘  │
└────────────────────────┬───────────────────────────────────────┘
                         │
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
        AWS           GCP           Azure
     (MSK/Kinesis) (Pub/Sub)   (Event Hubs)
```

### The Flow

1. **You write YAML** (or use the `sp` CLI) to describe your desired streaming state — environments, deployments, pipelines, and connections.
2. **STREAMINGPLUS validates** your manifests against the policy engine and schema constraints.
3. **The GitOps Reconciler** watches your connected Git repository and applies changes automatically when new commits land.
4. **The sp-agent** running in your target cluster receives reconciled state over a secure mTLS gRPC channel and provisions the actual cloud resources.
